Práctica para proceso de selección de Desarrollador
AlMaximo Consultoría TI

Práctica Desarrollo
Contexto Supuesto
Somos una organización transnacional de gran tamaño, comprometida con el
bienestar de nuestros empleados. Entre los beneficios que ofrecemos, se encuentran
préstamos que pueden ser pagados mediante abonos vía nómina o con reembolsos
en efectivo. Estos préstamos conllevan un pequeño interés, que varía según cada
caso.
Actualmente, el control de estos préstamos se realiza mediante hojas de cálculo en
Excel, lo que ha generado problemas en la precisión de la información. En varias
ocasiones, se ha proporcionado información errónea o los abonos se han realizado
en fechas incorrectas. Esto ha llevado a la gerencia a determinar la necesidad de
implementar un sistema robusto que permita un mejor control de los préstamos y sus
pagos.
Se espera que el desarrollo del sistema se lleve a cabo en fases, permitiendo una
implementación gradual y controlada que asegure la correcta y cada vez mejor gestión
de los préstamos.
Requerimiento Funcional
Desarrollar un pequeño sistema Web que permita dar de alta préstamos para cada
empleado, los datos que debe tener el préstamo son: IdPrestamo, Empleado,
CantidadTotalPrestada, CantidadTotalAPagar(incluyendo intereses),
InterésAprobado, interésMoratorio, TipoPagosAbonos (Semanal, Catorcenal,
Quincenal, Mensual, AlFinal), FechaPrimerPago (No aplica para AlFinal),
TotalAbonadoCapital, TotalAbonadoIntereses, Saldo, FechaFinalPago, Autor-
PersonaQueAutoriza (también son empleados), Notas (comentarios).
Es decir, para esta fase, solo se considera una pantalla para administrar los
prestamos donde se incluyan funcionalidades de Buscar (Listar), Agregar, Actualizar
y Eliminar Prestamos.
Se debe guardar un histórico de los cambios que se hacen cada vez que el préstamo
se actualiza, incluso cuando se crea, esto debido a que, en caso de ser necesario
tener la capacidad de consultar o saber quién, cuando y que cambio en cada
préstamo. Aunque en esta fase, solo se necesita visible o accesible la información
más reciente.
Consideraciones adicionales:
✓ Los Empleados (id, numnomina, nombres, apellido1, apellido2),
TipoPagosAbonos (id, nombrecorto, descripción), EmpleadosAutorizadores
solo estarán en un ámbito de base de datos, en tablas.
✓ Los formatos de fechas deben ser lo más cercano a “dd/mes/aaaa”
(16/ago/2024).
✓ Las cantidades deben considerar 2 decimales.
✓ EmpleadosAutorizadores pueden estar en un listado de selección, ya que son
3-5 posibles.
✓ Debido a que los empleados pueden ser muchos, considere una forma eficaz
y practica de seleccionarlo al asignarlo al préstamo.
Requerimientos Técnicos
✓ Base de Datos SQL Server 2017 o superior (Express Edition)
✓ Toda consulta o sentencia a BD debe hacerse con procedimientos
almacenados.
✓ C# con Net Core (6 superior)
✓ Modelo MVC Con servicios Rest Full (BackEnd) o con Repositorios
✓ JavaScript, JQuery, llamadas Ajax, Bootstrap o algún FW como Angular o
Vue.
Valorables
✓ Funcionalidades no especificadas que mejoren la experiencia de usuario.
✓ Proyecto y código bien estructurado, optimizado, mantenible, limpio y legible.
✓ Seguimiento a estándares, patrones y buenas prácticas.
